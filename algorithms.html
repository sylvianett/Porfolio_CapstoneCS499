This artifact represents my Algorithms & Data Structures enhancement for the CS 499 Capstone. I selected my CS 320 testing/validation project (Appointment Service + JUnit testing) because it demonstrates structured data handling, repeatable validation logic, and algorithmic improvement through data-driven execution.

Category: Algorithms & Data Structures Course: CS 320 (Enhanced for CS 499) Milestone: 3

Enhancement Summary (Milestone Three)

For this enhancement, I improved maintainability and scalability by reducing repetitive test logic and applying a more algorithmic, structured execution approach. Instead of writing nearly identical test cases, I organized input cases into collections and executed validations using consistent iteration patterns.

This supports stronger software engineering practices because new test cases can be added simply by adding a new entry into the dataset rather than rewriting new test methods.

Reflection

Enhancing this artifact reinforced that testing is more than writing assertions—it involves designing a repeatable, scalable process that can grow as requirements expand. One challenge was ensuring the parameterized approach still clearly mapped back to original requirements without becoming too abstract. I addressed this by keeping datasets organized by requirement area and maintaining clear naming and strong validation logic.
